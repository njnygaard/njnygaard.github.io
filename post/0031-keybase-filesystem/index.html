Keybase Filesystem

17 Apr 2018, 14:50
cryptography / calibre / linux / mac / storage

Keybase has a great feature called kbfs. You get 10 Gigs of storage forever. They say that it will be free going forward, and I would like to believe them because I don’t think Keybase will ever really be popular. I would love to eat those words.

They mention that you should not expect your files to be resilient; this is an Alpha service, so there is no guarantee of safety. That got me thinking about use-cases. For a while I’ve been wondering where to store my Library for Calibre. I don’t want to violate ToS by storing it somewhere (even though the books are all mine), but I would like to distribute access to the library between my computers. I’ve recently run out of promotional space on Google Drive, so finding a cryptographically secure, free 10G of space on the Internet is appealing.

I have to keep in mind that I might lose the data at any time. This isn’t really such a bad thing because they’re just books. I would like to keep my library going, but I won’t be heart broken if I have to start again.

After you have Keybase, just make sure that Calibre is going to use a folder in that mount point for it’s library. If you already have Calibre installed, you can run the Wizard again to configure the startup directory with a new library.

Caveat

It is important to note that the Keybase filesystem requires and Internet connection to work. There is no sync model built in to the system. This means that you do not have local copies of the files.
